# 사용법 가이드 (HOW TO USE)

돈복사기(money-printer)는 **개인용 로컬 금융 터미널 + 페이퍼 자동매매 연구 도구**입니다.
아래는 처음 켜는 것부터 예시 전략을 페이퍼로 돌려보는 것까지의 기본 절차입니다.

> ⚠️ **수익을 보장하지 않습니다.** 모든 실행은 모의/페이퍼입니다. 백테스트 수치는 과거 데이터 기반이며
> 미래 수익을 약속하지 않습니다. 거래비용·세금·슬리피지·시장 위험으로 실제 결과는 더 낮을 수 있습니다.
> 이 도구는 매매 권유가 아니라 학습·기록용입니다.

---

## 1. 설치 & 실행

사전 준비: **Python 3.11+**, **Node.js 18+** (python.org, nodejs.org).

1. `start.bat` 더블클릭 (또는 PowerShell에서 `./start.ps1`).
2. 스크립트가 자동으로: 가상환경 생성 → 백엔드 의존성 설치 → 프론트 빌드 → `http://127.0.0.1:8000` 서빙 → 브라우저 오픈.
3. 종료: 실행 중인 터미널에서 `Ctrl+C`.

> 루프백(`127.0.0.1`)에만 바인딩되어 외부 기기에서 접근할 수 없습니다.

## 2. 첫 실행 — 마스터 비밀번호 + 온보딩

1. 첫 화면에서 **마스터 비밀번호 설정**. 이후 모든 화면은 잠금 해제 후에만 보입니다.
2. 온보딩 마법사: 앱 소개 + 위험 고지 → (선택) Gemini 키 → (선택) DART 키 → 완료.
3. 키는 전부 선택 사항입니다. 키 없이도 시세·차트·뉴스·백테스트·페이퍼 자동매매가 동작합니다.

> 비밀번호는 복구 불가입니다. 분실 시 `.data` 폴더를 지우고 처음부터 다시 설정해야 합니다(저장 데이터 삭제).

## 3. 기본 탭 둘러보기

- **마켓 / 차트**: 시세, 풀 인터랙티브 차트(지표·그리기 도구).
- **포트폴리오**: 보유 종목 원화 환산 비중·손익, 자동투자 내역.
- **리서치 / 뉴스 / 공시**: 뉴스, SEC·DART 공시, 옵션.
- **수동 주문**: 내부 모의 또는 KIS 모의투자로 직접 주문.
- **자동 / 전략 빌더**: 전략 생성·백테스트·자동 실행.

## 4. 전략 빌더 — Example 1로 시작

신규 사용자는 비활성 예시 전략 **`Example 1 — SMA 골든크로스`** 가 자동으로 1개 들어 있습니다.

**Example 1 구성**
- 진입: `SMA20`이 `SMA60`을 상향 돌파 (골든크로스)
- 청산: `SMA20`이 `SMA60`을 하향 돌파 (데드크로스)
- 손절 −7% / 익절 +15%
- 타임프레임: 일봉 · 대상: `AAPL`

### 직접 만들기
1. 전략 빌더 탭 → **+ 새 전략(조건형)**.
2. **진입 조건**을 추가(전부 AND). 예: `RSI(14) < 30` + `종가 > SMA120`.
3. **청산 조건**(선택) + 손절/익절 % 입력.
4. 대상 종목·타임프레임 선택 후 **저장**.

> 사용 가능한 지표·연산자 목록은 빌더 화면의 드롭다운에 표시됩니다(이동평균·RSI·MACD·볼린저·일목·스토캐스틱 등, AND 결합).

## 5. 백테스트로 검증

1. 전략 카드에서 **백테스트** (또는 편집 화면에서 종목·기간 선택 후 실행).
2. 결과: 수익률 / 매수후보유 대비 / 최대낙폭(MDD) / 거래수 / 승률.
3. 백테스트는 **룩어헤드를 제거**(신호=확정봉, 체결=다음봉 시가)하고 **수수료·슬리피지**와 한국 종목 **매도세(0.18%)** 를 반영합니다 — 그래서 보수적으로 나옵니다.

> 백테스트가 좋아도 미래 보장이 아닙니다. 과최적화를 경계하세요.

## 6. 페이퍼 자동 실행

1. 검증이 끝난 전략 카드의 **ON** 토글.
2. **지금 1회 실행** → 현재 신호로 페이퍼 주문이 기록됩니다.
3. **자동 실행 ON** → 앱이 켜져 있는 동안 주기적으로 평가·체결(장중 게이트 적용).
4. 결과는 **포트폴리오 / 리스크 탭**에서 추적됩니다.

**강제 리스크 가드(전략이 바꿀 수 없음)**
- 당일 −5% 도달 → 신규 매수 금지
- 전체 낙폭 −20% 도달 → 자동전략 정지
- 단일 종목 비중·현금 하한·롱온리·레버리지/인버스 금지
- 데이터가 없거나 지연이면 주문 차단/표시

> 자동 실행은 **앱이 켜져 있는 동안만** 동작합니다. 로그인 시 자동 기동이 필요하면 `install-autostart.ps1`(작업 스케줄러)을 등록하세요.

## 7. 체결 방식 — 내부 모의 vs KIS 모의투자

설정 탭에서 `broker_mode` 선택:
- **`paper_internal`**(기본): 앱 내부 PaperBroker가 즉시 기록. 키 불필요.
- **`kis_mock`**: 한국투자증권 OpenAPI **모의 도메인**으로 주문 전송(국내 시장가 / 해외 지정가). KIS 앱키·시크릿·모의 계좌번호 필요. **실거래 아님.**

## 8. 안전 수칙

- 실거래는 기본 비활성(`LIVE_TRADING_ENABLED=false`)이며 이 도구는 의도적으로 실거래 경로를 막아 둡니다.
- `.env`, `.data/`(DB·비밀키)는 절대 공유/커밋하지 마세요(`.gitignore`로 제외됨).
- 백테스트·페이퍼 결과를 실현 수익으로 단정하지 마세요. 소액·모의로 충분히 검증한 뒤에만 판단하세요.
- 투자 책임은 본인에게 있습니다. 이 앱은 정보·학습 도구입니다.
